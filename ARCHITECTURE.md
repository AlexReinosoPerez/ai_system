## ğŸ” Key Design Decisions

### Why Scoped Workspace?
- **Isolation**: External tools only see allowed_paths
- **Security**: Cannot accidentally modify unrelated files
- **Clarity**: Explicit scope of changes

### Why MD5 Hashing?
- **Deterministic**: Same content = same hash
- **Independent**: Works regardless of timestamps
- **Fast**: Quick for small-to-medium files

### Why Append-Only Reports?
- **Audit trail**: Never lose history
- **Simple**: No complex update logic
- **Debuggable**: Can trace all executions

### Why No Commits?
- **Control**: Human decides when to commit
- **Review**: Can inspect changes before committing
- **Safety**: Prevents accidental commits
